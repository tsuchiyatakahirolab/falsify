# Scope Lock — Build Week MVP

## Must ship

1. Public web application.
2. Input via pasted text plus at least one of URL or document upload.
3. Atomic claim extraction with claim types.
4. Explicit "what would make this claim wrong?" falsification questions.
5. Separate supporting-evidence and contradictory-evidence retrieval.
6. Source provenance visible to the user.
7. Qualified verdict ontology.
8. At least basic inference-overreach audit.
9. At least one deterministic or semi-deterministic audit among date, number, or metadata checks.
10. Evidence Map / Claim Card UI.
11. "Challenge this finding" adversarial re-check.
12. One compelling public-source China-to-Japan narrative demonstration.
13. Public GitHub repository with license and reproducible setup.
14. Public judge-ready demo without rebuilding.
15. Build Week README and video requirements.

## Should ship if time remains

- citation-to-claim fit when citations are directly extractable;
- document PDF ingestion;
- bilingual English/Japanese UI or output;
- exportable Rebuttal Card;
- saved local session in browser;
- small human feedback mechanism.

## Explicitly deferred

- user accounts;
- billing;
- team workspaces;
- enterprise SSO;
- permanent manuscript library;
- full systematic-review workflow;
- comprehensive citation graph;
- automated journal peer review;
- full ML/QML experimentation;
- proprietary DB connectors;
- full CONPD/JNOM integration;
- all-language localization;
- comprehensive browser extension;
- mobile native app;
- global misinformation monitoring dashboard.

## Scope-change rule

A new feature may enter the MVP only if:
- it directly improves judging criteria or the three-minute demo;
- it replaces an existing lower-value feature;
- it does not threaten public deployment, evals, privacy, or submission completeness.
