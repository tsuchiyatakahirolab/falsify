# Public Demo Script — 2:40 Target

The final video must be public on YouTube, under three minutes, and include audio explaining both Codex and GPT-5.6 use.

## 0:00–0:18 — The problem

**Screen:** Falsify landing page.

**Voiceover:** “A claim can cite real facts and still reach a conclusion those facts do not establish. Most AI tools help an argument sound stronger. Falsify is built to look for how it could fail.”

## 0:18–0:38 — The product

**Screen:** Scroll to the input workbench; point to the method labels.

**Voiceover:** “Falsify decomposes an argument into atomic claims, asks what would make each one wrong, searches separately for support and contradiction, audits citations and inference, and keeps uncertainty visible. There is no global truth or credibility score.”

## 0:38–1:05 — Load the flagship demo

**Screen:** Select **Load flagship public-source demo**; show the four-claim map.

**Voiceover:** “This public-source demo tests a China-to-Japan defense narrative. The first claim is a supported fact about Japan's official draft budget proposal and planned capabilities—not final Diet enactment. The second is a much stronger historical analogy. The third and fourth are reciprocal claims about deceptive intent.”

## 1:05–1:35 — Inspect the evidence chain

**Screen:** Open/scroll Claim 2. Show the falsification questions, both evidence columns, source links, issue labels, and unresolved items.

**Voiceover:** “Falsify preserves the real proposal and planned capability expansion as legitimate grounds for concern, while showing that planned expenditure growth alone does not establish a revival of pre-1945 militarism. Every source keeps its publisher, date, URL, relevance, directness, and provenance note.”

## 1:35–1:55 — Symmetric standard

**Screen:** Briefly show Claims 3 and 4 and both `ATTRIBUTION NOT ESTABLISHED` labels.

**Voiceover:** “The same standard applies in both directions. A Chinese attribution of Japanese intent and a Japanese attribution of Chinese intent each need independent evidence. Disagreement is not proof of deception.”

## 1:55–2:18 — Challenge Falsify

**Screen:** On Claim 2 select **Challenge this finding**; show `QUALIFIED`, original/revised verdicts, and the National Diet source.

**Voiceover:** “Falsify is itself falsifiable. The re-check finds a Japanese parliamentary research record and qualifies the analysis: narrower concern about changing defense restraint is legitimate, while the strongest historical analogy remains unestablished.”

## 2:18–2:35 — Codex, GPT-5.6, and the public runtime

**Screen:** Show a brief architecture graphic or README section.

**Voiceover:** “Codex was the primary build environment for the TypeScript architecture, UI, tests, security hardening, evals, and release package. The repository includes a GPT-5.6 Responses API path with Structured Outputs, citation allowlisting, synthesis, and re-check. Because Build Week credits ran out, this public zero-cost runtime transparently uses Gemini grounding for its two live searches and never labels that output GPT-5.6.”

## 2:35–2:40 — Close

**Screen:** Return to the Evidence Map title.

**Voiceover:** “Don't trust the answer. Inspect its evidence.”

## Recording checklist

- Use the deployed public URL, not localhost.
- Keep the cursor deliberate and text legible at 1080p or higher.
- Record system audio/voice clearly; use no copyrighted music.
- Do not show API keys, browser secrets, personal tabs, or private material.
- Keep the final cut below 2:55 to leave upload/transcoding margin.
- Verify the final YouTube link is public in a signed-out window.
